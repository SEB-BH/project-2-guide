### Objective

Apply what you’ve learned about RESTful routing by designing a routing chart for a new resource such as **"songs"** or **"recipes"**.

### Instructions

1. **Choose your resource:**  
   Feel free to be creative.

2. **Identify the CRUD operations:**  
   Think about the operations you would need to perform on your resource:

   - **Create:** Add new data.
   - **Read:** Retrieve existing data (all songs or a specific song).
   - **Update:** Modify existing data.
   - **Delete:** Remove data.

3. **Map CRUD to routes:**  
   For each operation, define:

   - The **HTTP method** (`GET`, `POST`).
   - The **endpoint** or path (`/songs`, `/songs/:id`).
   - A brief description of the route’s purpose.

4. **Create your chart:**
   - **Option 1:** Draw the chart by hand.
   - **Option 2:** Use a diagramming tool such as **Whimsical**, **Lucidchart**, or even a spreadsheet application.
